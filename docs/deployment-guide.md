# คู่มือการติดตั้งระบบ orderstock (ใบออเดอร์สินค้า)

คู่มือนี้สำหรับผู้ดูแลระบบ/ฝ่ายไอทีของลูกค้า สำหรับติดตั้งและเปิดใช้งานระบบ orderstock บนเครื่อง Windows ที่เชื่อมต่อกับ Microsoft SQL Server

> หมายเหตุสำคัญ: เวอร์ชัน SQL Server ของลูกค้ายังไม่ได้รับการยืนยัน ระบบรองรับ **SQL Server 2017 ขึ้นไป** (2016 อาจใช้ไม่ได้) กรุณายืนยันเวอร์ชันก่อนติดตั้งจริง แล้วปรับค่า `COMPATIBILITY_LEVEL` ในสคริปต์ `db/create-database-and-login.sql` ให้ตรง (140 = 2017, 150 = 2019, 160 = 2022)

---

## ⚠️ คำเตือนสำคัญ: `db_TCL` คือฐานข้อมูล ERP จริงของลูกค้า

**ฐานข้อมูล `db_TCL` บนเซิร์ฟเวอร์ SQL Server ของลูกค้า ไม่ใช่ฐานข้อมูลเฉพาะของ orderstock
— แต่เป็นฐานข้อมูล ERP/บัญชีที่ใช้งานจริงของลูกค้า** ซึ่งมีตารางอื่นอีกหลายร้อยตารางที่ไม่เกี่ยวกับ
orderstock (เช่น Customer, Supplier, SalesInvoiceHdr/Dtl, InventoryItem, AccountChart ฯลฯ)
ตาราง 9 ตารางของ orderstock อยู่ร่วมในฐานข้อมูลเดียวกันนี้ **ห้ามแตะต้องตารางอื่นนอกเหนือจาก
9 ตารางของ orderstock โดยเด็ดขาด**

**กฎความปลอดภัยที่ต้องปฏิบัติตามเสมอ:**

1. **ห้ามรัน `prisma migrate reset`, `prisma migrate dev`, หรือ `prisma db push --force-reset`
   กับ `db_TCL` โดยเด็ดขาด** — คำสั่งเหล่านี้จะ **ลบตารางทั้งหมด** ในฐานข้อมูล ซึ่งหมายถึงข้อมูล ERP
   ทั้งระบบของลูกค้าจะหายไปด้วย ไม่ใช่แค่ตารางของ orderstock
2. **ห้ามรันสคริปต์ `db/create-database-and-login.sql` ซ้ำกับเซิร์ฟเวอร์จริงอีก** — สคริปต์นี้ถูกใช้ไปแล้วครั้งเดียว
   ตอนติดตั้งครั้งแรก (ดูคำเตือนภายในไฟล์สคริปต์)
3. **ห้ามแก้ไข `COMPATIBILITY_LEVEL`** ของฐานข้อมูล `db_TCL` — ฐานข้อมูลนี้ตั้งไว้ที่ 130 อยู่แล้ว
   และแอปทำงานได้ปกติที่ระดับนี้ การเปลี่ยนค่านี้จะกระทบ query plan ของระบบ ERP ทั้งระบบของลูกค้า
   ไม่ใช่แค่ orderstock
4. การใช้งานแอปตามปกติ (เพิ่ม/แก้ไขร้านค้า สินค้า บันทึกออเดอร์) และการ seed ข้อมูลเริ่มต้น
   (`prisma/seed.ts`) เป็นการดำเนินการที่ปลอดภัย เพราะแตะเฉพาะ 9 ตารางของ orderstock เท่านั้น

รายละเอียดทางเทคนิคเพิ่มเติม: `process/context/database/all-database.md` หัวข้อ
"Production DB: shared ERP database db_TCL — DANGER guardrails"

## 1. สิ่งที่ต้องเตรียม (Prerequisites)

- **Windows** (เครื่องเซิร์ฟเวอร์หรือเครื่องที่จะรันแอป)
- **Node.js LTS** (เวอร์ชัน 22 ขึ้นไป) — ดาวน์โหลดจาก https://nodejs.org
- **pnpm** — ติดตั้งด้วยคำสั่ง: `npm install -g pnpm`
- **Microsoft SQL Server 2017 ขึ้นไป** พร้อมสิทธิ์ผู้ดูแล (เช่นล็อกอิน `sa`)
- **SSMS** (SQL Server Management Studio) หรือ `sqlcmd` สำหรับรันสคริปต์ SQL
- **เว็บเบราว์เซอร์ Chrome หรือ Microsoft Edge** (จำเป็นสำหรับการพิมพ์ให้ตรงแบบฟอร์ม)

---

## 2. ติดตั้งไฟล์โปรแกรม

1. คัดลอกโฟลเดอร์โปรแกรมทั้งหมดไปยังเครื่องเซิร์ฟเวอร์ (เช่น `C:\apps\orderstock`)
2. เปิด Command Prompt / PowerShell ไปที่โฟลเดอร์นั้น แล้วรัน:

   ```
   pnpm install
   ```

---

## 3. สร้างฐานข้อมูลและผู้ใช้ (SQL Server)

รันสคริปต์ SQL **ตามลำดับนี้** ใน SSMS (เข้าสู่ระบบด้วยล็อกอินสิทธิ์ผู้ดูแล เช่น `sa`):

1. **`db/create-database-and-login.sql`** — สร้างฐานข้อมูล `orderstock`, ล็อกอิน `orderstock_app`, ผู้ใช้ และสิทธิ์
   - **สำคัญ:** แก้รหัสผ่านในบรรทัด `CREATE LOGIN ... WITH PASSWORD = N'REPLACE_WITH_A_STRONG_PASSWORD'` ให้เป็นรหัสผ่านที่ปลอดภัย **ก่อนรัน** (อย่าใช้ค่าตัวอย่าง)
   - ปรับ/เปิดใช้บรรทัด `COMPATIBILITY_LEVEL` ให้ตรงเวอร์ชัน SQL Server ของท่าน
2. **`db/create-orderstock-schema.sql`** — สร้างตารางทั้งหมด (เลือกฐานข้อมูล `orderstock` ก่อนรัน: `USE [orderstock];`)

> หมายเหตุสิทธิ์: สคริปต์ให้สิทธิ์ `db_owner` สำหรับการติดตั้งครั้งแรก หากต้องการจำกัดสิทธิ์แบบ least-privilege หลังสร้างตารางแล้ว ดูหมายเหตุในไฟล์สคริปต์ (เปลี่ยนเป็น `db_datareader` + `db_datawriter`)

---

## 4. ตั้งค่าไฟล์ `.env`

1. คัดลอกไฟล์ `.env.example` เป็น `.env`
2. แก้ค่าต่อไปนี้ในไฟล์ `.env`:

   - **`DATABASE_URL`** — สายการเชื่อมต่อฐานข้อมูล (รูปแบบ JDBC ของ Prisma) เช่น:
     ```
     DATABASE_URL="sqlserver://SERVERHOST:1433;database=orderstock;user=orderstock_app;password={รหัสผ่านที่ตั้งไว้};encrypt=true;trustServerCertificate=true"
     ```
     - อักขระพิเศษในรหัสผ่าน ( `: \ = ; / [ ] { }` ) ต้องครอบด้วยปีกกา เช่น `password={Pa:ss;word}`
     - ท่านสามารถตั้งค่านี้ผ่านหน้า "ตั้งค่าฐานข้อมูล" ในแอปได้ภายหลัง (ดูข้อ 8)

   - **`AUTH_SECRET`** — คีย์สำหรับเซ็นเซสชันเข้าสู่ระบบ สร้างด้วยคำสั่ง:
     ```
     openssl rand -base64 32
     ```
     (บน Windows ที่ไม่มี openssl ใช้ Git Bash หรือ `node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"`)

   - **`AUTH_TRUST_HOST=true`** — จำเป็นสำหรับการรันแบบ self-hosted

   - **`SEED_ADMIN_PASSWORD`** — รหัสผ่านผู้ดูแลระบบเริ่มต้น (ใช้ครั้งเดียวตอน seed) ตั้งเป็นรหัสผ่านที่ปลอดภัย

---

## 5. สร้างผู้ดูแลระบบเริ่มต้น (Admin seed)

รันคำสั่ง (ครั้งเดียว):

```
pnpm tsx prisma/seed.ts
```

ระบบจะสร้างบัญชีผู้ดูแลระบบเริ่มต้นจากค่า `SEED_ADMIN_PASSWORD` (หากไม่ได้ตั้ง ระบบจะสุ่มรหัสผ่านและแสดงบนหน้าจอ — จดไว้) **เปลี่ยนรหัสผ่านหลังเข้าสู่ระบบครั้งแรก**

---

## 6. สร้างและรันแอป (พร้อมตั้งเป็นบริการ Windows)

1. Build แอป:
   ```
   pnpm build
   ```
2. ทดสอบรัน:
   ```
   pnpm start
   ```
   แอปจะเปิดที่ `http://localhost:3000`

### ตั้งเป็นบริการ Windows ด้วย NSSM (แนะนำ — รอการยืนยันจากฝ่ายไอทีลูกค้า)

การตั้งเป็นบริการช่วยให้แอป **รีสตาร์ทอัตโนมัติ** เมื่อเครื่องรีบูตหรือเมื่อบันทึกการตั้งค่าฐานข้อมูลใหม่ (หน้าตั้งค่าจะปิดตัวเองแล้วให้บริการเริ่มใหม่)

1. ดาวน์โหลด NSSM จาก https://nssm.cc
2. ติดตั้งบริการ:
   ```
   nssm install orderstock "C:\Program Files\nodejs\pnpm.cmd" start
   ```
   - ตั้ง **Startup directory** เป็นโฟลเดอร์โปรแกรม (เช่น `C:\apps\orderstock`)
   - ที่แท็บ **Exit actions** ตั้งให้ **Restart** เมื่อโปรแกรมปิดตัว (สำคัญ — ทำให้การบันทึกการตั้งค่าฐานข้อมูลแล้วรีสตาร์ททำงานอัตโนมัติ)
3. เริ่มบริการ: `nssm start orderstock`

> **ทางเลือก (IIS):** หากลูกค้าใช้ IIS อยู่แล้ว สามารถตั้ง reverse proxy (ARR) ชี้ไปที่ `http://localhost:3000` ได้ แต่ยังต้องมีตัวจัดการให้แอป Node รีสตาร์ทเอง (เช่น NSSM หรือ pm2-windows) เพื่อให้การบันทึกการตั้งค่ารีสตาร์ทได้

> **หากไม่ใช้ NSSM (ไม่มีการรีสตาร์ทอัตโนมัติ):** หลังกด "บันทึกและรีสตาร์ท" ในหน้าตั้งค่า แอปจะปิดตัวเอง ต้องเริ่มแอปใหม่ด้วยมือ (`pnpm start`)

---

## 7. ตรวจสอบสถานะระบบ (Health check)

เปิด `http://localhost:3000/api/health` — หากเชื่อมต่อฐานข้อมูลได้จะแสดง `{"ok":true}`

หากได้ `{"ok":false}` ให้ตรวจสอบ `DATABASE_URL`, สถานะ SQL Server, และไฟร์วอลล์พอร์ต 1433

---

## 8. เปลี่ยนการเชื่อมต่อฐานข้อมูลผ่านหน้าแอป (ผู้ดูแลระบบเท่านั้น)

1. เข้าสู่ระบบด้วยบัญชี **ADMIN** แล้วคลิกเมนู **"ตั้งค่าฐานข้อมูล"**
2. กรอกข้อมูลการเชื่อมต่อทีละช่อง (host, port, database, user, password ฯลฯ)
   - หรือ **วางสาย connection string** (ADO.NET หรือ `sqlserver://`) ในช่องด้านบนแล้วกด "เติมค่า" เพื่อเติมช่องอัตโนมัติ (ตรวจสอบก่อนบันทึกเสมอ)
3. กด **"ทดสอบการเชื่อมต่อ"** — ต้องขึ้น "เชื่อมต่อฐานข้อมูลสำเร็จ" ก่อน
4. กด **"บันทึกและรีสตาร์ท"** — ระบบจะสำรองไฟล์ `.env` เดิมเป็น `.env.bak` เขียนค่าใหม่ แล้วรีสตาร์ทเพื่อใช้ค่าใหม่ (ภายใต้ NSSM จะกลับมาเองใน ~2-5 วินาที)

> ความปลอดภัย: ระบบจะเขียนค่าใหม่ **ก็ต่อเมื่อทดสอบการเชื่อมต่อผ่านแล้วเท่านั้น** สายที่ผิดจะไม่ถูกบันทึกและไม่ทำให้แอปปิดตัว

---

## 9. คำแนะนำการพิมพ์ (สำคัญมากเพื่อให้ตรงแบบฟอร์ม)

- ใช้ **Chrome หรือ Microsoft Edge เท่านั้น** (เบราว์เซอร์อื่นเลย์เอาต์อาจเพี้ยน)
- ในหน้าต่างพิมพ์ (Print):
  - **Scale (มาตราส่วน) = 100%** (อย่าใช้ "Fit to page")
  - **ปิด Headers และ Footers** (เอาวันที่/URL ที่เบราว์เซอร์ใส่ให้ออก)
  - ขนาดกระดาษ **A4** แนวนอน (Landscape)
- **ทดสอบพิมพ์จริงที่หน้างานอย่างน้อยหนึ่งครั้ง** ก่อนใช้งานจริง เพื่อยืนยันว่าเครื่องพิมพ์ให้ขนาดตรงตามแบบฟอร์ม (ระยะเป็นมิลลิเมตร) — ความแม่นยำของเครื่องพิมพ์จริงยังไม่ได้ตรวจสอบที่หน้างาน

---

## 10. การสำรองข้อมูล (Backup)

- **ฐานข้อมูล:** สำรอง `orderstock` เป็นประจำด้วย SQL Server Backup (เช่น Maintenance Plan หรือ `BACKUP DATABASE`)
- **ไฟล์ตั้งค่า:** สำรองไฟล์ `.env` (และ `.env.bak` หากมี) ไว้ในที่ปลอดภัย — มีรหัสผ่านฐานข้อมูลและ `AUTH_SECRET` **อย่าเก็บไว้ใน source control หรือแชร์สาธารณะ**

## 11. การแก้ปัญหา (Troubleshooting)

| อาการ | วิธีแก้ |
|---|---|
| `/api/health` แสดง `ok:false` | ตรวจ `DATABASE_URL`, SQL Server รันอยู่หรือไม่, พอร์ต 1433 เปิดหรือไม่ |
| เข้าสู่ระบบไม่ได้ / หน้า login วนซ้ำ | ตรวจว่าตั้ง `AUTH_SECRET` และ `AUTH_TRUST_HOST=true` แล้ว |
| พิมพ์แล้วขนาดเพี้ยน | ตรวจว่า Scale = 100%, ปิด headers/footers, ใช้ Chrome/Edge |
| **ถูกล็อกเอาต์ทั้งระบบหลังบันทึกการตั้งค่าฐานข้อมูลผิด (Lockout recovery)** | ระบบตรวจการเชื่อมต่อก่อนบันทึกเสมอ แต่หากยังเข้าไม่ได้ (เช่นฐานข้อมูลล่มภายหลัง): แก้ด้วยมือ — เปิดไฟล์ `.env` แก้ `DATABASE_URL` ให้ถูก **หรือ** กู้คืนจากไฟล์สำรองด้วยคำสั่ง `copy .env.bak .env` (Windows) แล้วรีสตาร์ทแอป |

> การกู้คืนกรณีถูกล็อกเอาต์ **ทำที่ไฟล์ `.env` โดยตรงเท่านั้น** ระบบไม่มีทางลัดเข้าโดยไม่ผ่านการยืนยันตัวตน (เป็นการออกแบบเพื่อความปลอดภัย)
