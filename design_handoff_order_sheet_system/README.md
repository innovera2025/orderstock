# Handoff: ระบบใบออเดอร์สินค้ารายวัน (Daily Order Sheet System) — ยิ่งเจริญ

## Overview

ระบบทดแทน "ใบออเดอร์สินค้า" กระดาษของโรงงาน/ผู้ผลิตอาหาร: เสมียนกรอกจำนวนสินค้าที่แต่ละร้านค้า (~29 ร้าน) สั่งในแต่ละวันลงตารางขนาดใหญ่ (ร้าน × สินค้า 20 คอลัมน์), ระบบรวมยอดท้ายตาราง/ยอดผลิตอัตโนมัติ, จัดการร้านค้า/สินค้า/ผู้ใช้ และพิมพ์ใบงานออกกระดาษ A4 แนวนอนตามฟอร์มเดิมได้เป๊ะ. มีเวอร์ชันมือถือสำหรับพนักงานหน้างานกรอกทีละร้าน — ใช้ฐานข้อมูลชุดเดียวกับเดสก์ท็อป.

ต้นแบบข้อมูลทั้งหมดมาจากใบงานจริงวันที่ 13/3/69 สถานที่ "ยิ่งเจริญ" (ดู `reference/ใบงานต้นฉบับ 13-3-69.png`).

## About the Design Files

ไฟล์ใน `design_files/` เป็น **design reference ที่สร้างด้วย HTML** (รูปแบบ Design Component: template HTML + logic class ใน `<script>`) — เป็น prototype แสดง look & behavior ที่ตั้งใจ **ไม่ใช่โค้ด production ที่ให้ copy ไปใช้ตรง ๆ**. งานของคุณคือ **สร้างดีไซน์เหล่านี้ขึ้นใหม่ใน codebase จริง** ด้วย stack/แพตเทิร์นที่โปรเจกต์ใช้อยู่ (React/Next.js, Vue, Flutter ฯลฯ). ถ้ายังไม่มี codebase — แนะนำ Next.js (เว็บแอดมิน) + PWA หรือ Flutter (มือถือ) + PostgreSQL.

เปิดไฟล์ .dc.html ในเบราว์เซอร์ไม่ได้โดยตรงนอกสภาพแวดล้อมเดิม (พึ่ง `support.js` + โฟลเดอร์ `_ds/`) — ให้ **อ่านเป็นซอร์สอ้างอิง**: template ส่วนบนคือ markup + inline styles ทั้งหมด, class `Component` ส่วนล่างคือ business logic / state ครบถ้วน.

## Fidelity

**High-fidelity (hifi)** — สี ตัวอักษร ระยะ ขนาด และ interaction เป็นค่าตั้งใจสุดท้าย ให้ทำตามให้ตรง (pixel-perfect) โดยใช้ design tokens ใน `design_tokens/`.

ข้อยกเว้น: ข้อมูลย้อนหลังในหน้า "ประวัติออเดอร์" และบัญชีผู้ใช้เริ่มต้นเป็น **ข้อมูลสาธิต**; รหัสผ่าน "1234" เป็น demo เท่านั้น — production ต้องมี auth จริง (hash password, session/JWT).

## Design System (pguard)

ใช้ระบบ **pguard Design System** — โทน Deep Forest green, โครงสร้างจากเส้นขอบบาง 1px ไม่ใช่เงา. Tokens ฉบับเต็มอยู่ที่ `design_tokens/*.css` (colors / typography / spacing / semantic / fonts). ที่ใช้บ่อย:

- **สี**: brand anchor `#0E3B2E` (green-900), interactive `#1FA971` (green-500), hover `#15885B`, accent amber `#F59E0B`, danger `#E5484D`, พื้นแอป `#F6F8F7`, เส้นขอบ `#DEE5E2`, border-strong `#C6D0CC`, ตัวอักษรหลัก `#141A17`, muted `#69776F`, faint `#94A29D`, ไฮไลต์ช่องที่กรอก `#E9F6F0`
- **ฟอนต์**: IBM Plex Sans Thai (UI), IBM Plex Sans (Latin), IBM Plex Mono (ตัวเลข/ID/วันที่ ทั้งหมด + `font-variant-numeric: tabular-nums`) — โหลดจาก Google Fonts ได้ (`design_tokens/fonts.css`)
- **Radius**: การ์ด/แผง 11px, ปุ่ม/อินพุต 8px, ชิป/สถานะ pill (99px)
- **ปุ่ม**: primary = พื้น `#1FA971` ตัวขาว hover `#15885B` กดแล้วขยับลง 1px; secondary = พื้นขาว ขอบ 1.5px `#C6D0CC`; danger-ghost = ตัวแดง ขอบแดงจาง
- **Focus**: ขอบ `#1FA971` + วงแหวน `0 0 0 4px rgba(31,169,113,.45)`
- **Modal scrim**: `rgba(8,20,15,.5)` + `backdrop-filter: blur(3px)`
- **ไอคอน**: Lucide เท่านั้น (stroke 2px), ห้าม emoji
- Touch target มือถือ ≥ 44px

## Interactions & Behavior

- ปุ่มทั้งหมด: transition background 150ms; กดขยับลง `translateY(1px)`; แถวตาราง hover พื้น `#F6F8F7`
- Toast: กล่องลอยพื้น `#0E3B2E` ตัวขาว radius 8px กลางล่าง แสดง ~2.2s (มือถือ 1.8s) — ใช้ยืนยันทุก action (บันทึก/รีเซ็ต/ระงับ/เพิ่ม)
- Modal: fade + scale .96→1 200ms; คลิก scrim ปิด; คลิกการ์ดไม่ปิด (stopPropagation)
- ทุก state เปลี่ยน → บันทึกอัตโนมัติทันที (ใน prototype ใช้ localStorage; production = API + optimistic update)
- Validation: ช่องตัวเลขรับเฉพาะ `0-9.` (strip อักขระอื่น); ค่าว่าง = ลบรายการ; เพิ่มผู้ใช้/ร้าน ต้องมีชื่อ (และอีเมล) ไม่งั้น toast เตือน; ค่าจากปุ่ม − ไม่ต่ำกว่า 0

## State Management

หัวใจของระบบ (ดู logic class ใน prototype เป็น spec):

- `vals: {"<rowIdx>-<colIdx>": number}` — sparse map ของทุกช่องตาราง (แถว 0-28, คอลัมน์ 0-19)
- `notes: {rowIdx: string}` — หมายเหตุต่อร้าน
- `shopNames: string[29+]` — ชื่อร้านตามลำดับแถวใบงาน (ค่าว่าง = แถวว่าง, **index ห้ามเลื่อน**; เพิ่มร้าน = push ต่อท้าย)
- `prodNames: string[15]` — ชื่อกลุ่มสินค้า; mapping คอลัมน์→กลุ่ม: `COL_GROUP = [0,1,1,2,2,3,4,5,5,6,6,7,8,9,10,10,11,12,13,14]`; หน่วยย่อยต่อคอลัมน์: `['กก.','1 กก.','½ กก.','1 กก.','½ กก.','กระสอบ','กระสอบ','1 กก.','½ กก.','1 กก.','½ กก.','กก.','กก.','กป','หมู','ไก่','ปี๊บ','กก.','กก.','กก.']` (คอลัมน์ 0-15 หมวดสินค้า, 16-19 เครื่องปรุง)
- ค่าคำนวณสด (derived ทุกครั้งที่ vals เปลี่ยน): ยอดรวมต่อคอลัมน์, grand total, ยอดต่อร้าน, จำนวนร้านที่สั่ง (มีค่า>0 หรือมีหมายเหตุ)
- `weight`, `peep` (รวมน้ำหนัก กก. / รวมปี๊บ) — **ผู้ใช้กรอกเองเหมือนใบกระดาษ**; ใบงานจริง 13/3/69: 4,670 กก. / 163.00 ปี๊บ (ยอดหน่วยรวม 446). ถ้าธุรกิจให้สูตรน้ำหนักต่อหน่วยของแต่ละสินค้ามา ค่อยเปลี่ยนเป็นคำนวณอัตโนมัติ
- `users: [{id, name, email, roleLabel, active, last}]` + session ผู้ใช้ปัจจุบัน; login สำเร็จ → อัปเดตชื่อผู้บันทึก
- ตั้งค่า: `placeName`, `recorderName`, `recorderRole`, `hlFilled`
- ออเดอร์ผูกกับ (วันที่, สถานที่) — prototype มีวันเดียว; production ต้องเก็บเป็นเอกสารรายวัน + ประวัติจริง

ไฟล์ครบอยู่ที่ `design_tokens/` (colors.css, typography.css, spacing.css, semantic.css, fonts.css) — ใช้ค่าจากไฟล์เหล่านี้เป็น source of truth. สรุปที่ใช้จริงในดีไซน์อยู่ในหัวข้อ Design System ด้านบน.
